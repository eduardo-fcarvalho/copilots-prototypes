# 🤖 Copilots Prototypes

> Um framework de prompts para transformar IA em um agente técnico consistente, previsível e orientado por boas práticas.

---

## 🚀 Sobre o projeto

O **Copilots Prototypes** é um laboratório prático para construção de agentes de IA que vão além de respostas genéricas.

A proposta é simples:  
> Não apenas gerar código — mas controlar **como a IA pensa, decide e executa**.

Inspirado em copilotos modernos, este projeto define um conjunto de **modos operacionais** que tornam a IA mais próxima de um engenheiro de software consistente.

---

## 🎯 Problema

Ferramentas de IA costumam:

- Gerar respostas inconsistentes  
- Ignorar contexto de stack  
- Variar qualidade entre interações  
- Falhar em decisões arquiteturais  

---

## 💡 Solução

Este repositório introduz um sistema baseado em:

- Prompts estruturados  
- Regras de stack explícitas  
- Fluxo de decisão previsível  
- Estilo de resposta padronizado  

Resultado:

> Uma IA que responde de forma mais próxima de um desenvolvedor experiente.

---

## 🧩 Modos de operação

O sistema é dividido em quatro modos principais:

### 🤖 Agent — Execução

Resolve problemas diretamente.

- Geração de código consistente  
- Aplicação automática de boas práticas  
- Respostas estruturadas e objetivas  

👉 Use quando precisa **fazer**.

---

### 🧭 Plan — Estratégia

Define como resolver antes de implementar.

- Quebra em etapas  
- Identificação de riscos  
- Planejamento incremental  

👉 Use quando precisa **pensar antes de agir**.

---

### ❓ Ask — Diagnóstico

Evita decisões erradas por falta de contexto.

- Perguntas objetivas  
- Identificação de lacunas  
- Foco no desbloqueio  

👉 Use quando o problema **não está claro**.

---

### 📚 Study — Aprendizado

Explica conceitos com profundidade.

- Explicações estruturadas  
- Conexão entre teoria e prática  
- Adaptação ao contexto  

👉 Use quando quer **entender, não só executar**.

---

## 🧠 Diferenciais

- Consistência de respostas  
- Alinhamento com stack (Spring, Node, etc.)  
- Estrutura lógica (ação → justificativa → consequência)  
- Redução de ambiguidade  
- Melhor qualidade em decisões técnicas  

---

## 🎭 Estilo de resposta

As respostas seguem um padrão analítico inspirado na personagem EDI (Mass Effect):

- Linguagem objetiva e impessoal  
- Baseadas em lógica  
- Com justificativas técnicas claras  
- Focadas em consequência e impacto  

---

## ⚙️ Casos de uso

- Desenvolvimento backend (Node / Spring)  
- Padronização de respostas em times  
- Criação de copilotos internos  
- Estudos estruturados de tecnologia  
- Debugging guiado  

---

## 🛠️ Como usar

1. Escolha o modo (`Agent`, `Plan`, `Ask` ou `Study`)  
2. Forneça contexto (stack, problema, objetivo)  
3. Execute o prompt correspondente  
4. Itere conforme necessário
